Éditions et canaux

1. Un standard sous Linux

Pourvu que vous utilisiez une version assez récente de Linux, l’installation de Docker est un modèle de simplicité. Le but de la présente section n’est pas de détailler le processus pour toutes les distributions de Linux ni d’entrer dans des options complexes de quota sur la mémoire ou de paramétrage de sécurité, mais simplement de donner les instructions de base pour que le lecteur soit en mesure de réaliser les exemples qui suivront.

La page dédiée à l’installation de Docker sur le site de référence https://docs.docker.com/engine/installation/ est exhaustive sur le sujet, avec en particulier les options sur les différentes distributions Linux, les modes de mise en place sur les plateformes cloud, etc. De plus, Docker fait désormais partie des paquetages standards dans l’énorme majorité des distributions Linux et même des fonctionnalités activables sans installation sur les systèmes d’exploitation Windows. Il n’y a donc plus aucune difficulté pour installer Docker, si tant est qu’il y en ait eu un temps. Notre but, dans la présente section, est donc seulement de faire en sorte que le lecteur, qu’il souhaite utiliser une machine distante ou bien installer Docker sur son système d’exploitation Linux ou Windows, trouve rapidement la méthode la plus standard pour ce faire.

Le produit étant désormais disponible sous forme de plusieurs éditions, un point sera fait pour résumer les principales différences. Le mode de mise à jour de Docker étant une option à déterminer lors de l’installation, les différents canaux proposés par Docker pour cette opération seront également expliqués. Le choix du canal approprié permettra d’obtenir, en fonction de vos besoins, d’utiliser le plus tôt possible les nouvelles fonctionnalités ou, à l’inverse, de privilégier la stabilité de votre environnement.

2. Les canaux Canal

Le moteur Docker peut suivre différents modes de déploiement et de mise à jour en fonction du compromis souhaité par l’utilisateur entre la rapidité de mise à disposition des nouveautés et le niveau de stabilité voulu. Les modes disponibles sont les suivants :

  • Stable : il s’agit du mode à recommander par défaut, disponible largement et avec les fonctionnalités stabilisées.

  • Test : ce mode donne accès aux fonctionnalités de pré-release du produit. Il est donc un peu moins stable que le précédent mode et il ne doit pas être utilisé en production, mais il ne s’agit pas non plus de fonctionnalités en cours de développement, donc le niveau d’utilisabilité est tolérable.

  • Nightly : ce dernier mode est à réserver aux développeurs ou aux personnes suivant activement le développement de Docker, car il contient les fonctionnalités en cours de programmation. Bien sûr, les fonctionnalités existantes ne sont pas censées être impactées, mais ce mode ne donne aucune garantie sur la stabilité. Dans ce mode, la mise à jour de Docker se fait au jour le jour.

Dans le doute, choisissez un canal de diffusion de type Stable.

3. Les éditions Édition

Attention, ces canaux ne doivent pas être confondus avec la version de Docker, qui peut être une Community Edition (gratuite, support par la communauté, donc sans garantie de correction ni même de réponse) ou une Enterprise Edition (avec du support engageant, et donc payante). Dans les exemples ci-dessous, nous ciblons presque systématiquement une version CE, de façon à être sûr que tous les lecteurs soient en mesure de les réaliser. Seule l’installation sous Windows Server se fera sur une Enterprise Edition, car il s’agit de l’édition déployée par le mode d’installation démontré, mais cela n’aura pas d’impact sur le contenu de la section correspondante.

L’Enterprise Edition peut être utilisée en mode d’essai, avec toutefois une durée limitée pendant laquelle les fonctionnalités additionnelles par rapport à la Community Edition sont disponibles. Le présent ouvrage ne montrera aucune des fonctionnalités spécifiques de l’Enterprise Edition, son orientation étant plus de donner une approche la plus pédagogique et complète possible de la plateforme et des concepts de Docker, et non de viser une exhaustivité en termes de fonctionnalités présentées.
