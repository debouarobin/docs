Utiliser des machines dans le cloud

Pour le lecteur qui souhaite ne rien avoir à installer sur sa machine et qui dispose d’un compte sur un service de cloud, l’approche la plus simple et la plus rapide pour disposer d’une installation de Docker fonctionnelle et réaliser les exercices à suivre est de créer une ressource dans le cloud.

Les services gérés de déploiement industriel de conteneurs seront étudiés en fin de livre, mais pour l’instant, ces approches ne sont pas utiles car l’objectif actuel est de bien comprendre le fonctionnement de Docker. Ceci implique en particulier de faire manipuler le client en ligne de commande, connaître son fonctionnement et ses options, etc. Or, toutes ces manipulations seraient cachées par la surcouche proposée par une éventuelle plateforme de gestion en production. Les services les plus simples des clouds seront donc montrés ci-dessous de façon à créer rapidement une machine, et ce pour un coût qui sera normalement nul, les offres gratuites suffisant pour un usage raisonné tel que le nécessitent les exercices à suivre.

Les lecteurs sachant déjà créer une machine virtuelle dans le cloud peuvent sauter la prochaine section et directement aller à la suivante pour voir comment déployer Docker sur leur machine virtuelle.

d. Gestion de la consommation

Afin d’éviter les surprises liées à une surconsommation de ressources, l’offre gratuite nécessitant tout de même de fournir un numéro de carte bancaire, il est recommandé de mettre en place quelques sécurités.

La première consiste à paramétrer des alertes pour arrêter automatiquement la machine dans certaines conditions. La documentation mise à disposition sur https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/UsingAlarmActions.html explique comment faire pour réagir à une inactivité d’une certaine durée. Cette méthode est toutefois largement limitée par le fait qu’une machine faisant tourner une application de type serveur peut très bien ne jamais être complètement inactive. D’autres solutions existent, qui utilisent le mécanisme Lambda pour réagir à des triggers temporels, mais ne sont pas suffisamment simples pour être rapidement montrées dans le cadre de ce livre.

Une seconde sécurité consiste à mettre en place des budgets pour contrôler les coûts. Là encore, cette solution est limitée car elle ne rend pas possible de limiter de manière stricte les coûts avec un maximum indépassable, bien que cette demande soit très populaire sur les forums. La page https://aws.amazon.com/fr/getting-started/hands-on/control-your-costs-free-tier-budgets/ est le point de départ pour les bonnes pratiques de limitation de coût. La première action à mettre en place est sur l’offre gratuite elle-même, le cas échéant.

 Allez sur la page : https://console.aws.amazon.com/billing/home#/

 Cliquez sur Préférences de facturation.

 Cochez l’option Recevoir les alertes d’utilisation de l’offre gratuite.

 Saisissez éventuellement une adresse mail alternative si vous ne souhaitez pas recevoir ces alertes sur l’adresse ayant servi à la création du compte AWS.

 Créez également un budget de consommation avec des alertes comme montré dans la documentation citée plus haut.

Le temps de création de la machine peut être utilisé pour transformer le format de la paire de clés. Cette manipulation est réalisée dans l’utilitaire PuTTYgen, qui doit être installé au préalable si ce n’est pas le cas.

 Lancez PuTTYgen.

 Ouvrez le menu Conversions.

 Sélectionnez la commande Importkey.

d. Suppression de la machine virtuelle

Si vous souhaitez supprimer tout ce qui a été réalisé sur Azure pour être sûr de ne plus avoir à payer quoi que ce soit en lien avec ce qui a été installé plus haut, la manipulation est très simple :

3. Google Cloud Platform Google Cloud Platform

a. Offre

L’activation d’un compte gratuit sur le cloud de Google se fait à l’adresse https://console.cloud.google.com/freetrial. Comme pour les autres fournisseurs vus plus haut, l’utilisation d’une carte bancaire est nécessaire, même si aucun frais ne sera prélevé lors de l’abonnement (un mouvement très limité est toutefois réalisé, avec remboursement immédiat, de façon à valider effectivement le compte).

Le détail du contenu de l’offre gratuite est à chercher sur une autre URL, à savoir https://cloud.google.com/free.

L’offre est taillée globalement comme celle d’Amazon, avec un crédit équivalent à une toute petite machine. Dans la nomenclature de Google, il s’agit d’une machine de type f1-micro.

Enfin, comme les autres fournisseurs de Cloud, Google Cloud Platform propose un ensemble de services qui restent gratuits sans durée limite, contrairement à l’offre d’essai.

Attention toutefois au fait que ces services en eux-mêmes sont effectivement gratuits, mais pas les ressources sur lesquelles ils s’appuient. Il est ainsi possible de disposer d’un cluster Kubernetes de n’importe quelle taille, mais les nodes seront bien sûr des machines sur GCP, et donc avec un coût de facturation associé. Il convient donc, comme d’habitude, de bien vérifier sa consommation et en particulier à la fin de la période d’essai gratuite, de surveiller les coûts associés au passage éventuel à l’offre payante.

d. Suppression de la machine virtuelle

Lorsque vous souhaitez supprimer toutes les ressources afin de ne plus être facturé, l’opération la plus simple est de supprimer le projet lui-même :

 Cliquez sur le nom du projet en haut de l’interface.

 Dans la fenêtre modale qui apparaît, cliquez sur l’icône représentant un dossier avec une roue dentée juste à gauche de la commande Nouveau projet utilisée plus haut.

 Dans l’interface de gestion des ressources qui apparaît, sélectionnez le projet à supprimer avec la case à cocher et cliquez sur Supprimer, ou bien cliquez sur les trois petits points verticaux sur la droite du projet et lancez la même commande Supprimer.

 Une confirmation forte est également nécessaire. Pensez à bien saisir l’identifiant du projet et pas son nom, pour pouvoir valider sa suppression.

 Cliquez sur Arrêter.

Pour sécuriser encore plus, GCP ne supprime pas immédiatement le contenu complet :
